         directly producing structured JSON output.

Feature 1: Drug Normalization (NEW FILE: drug_normalizer.py)

File: drug_normalizer.py (NEW -- did not exist before)
Size: ~18KB | Catalog: 200+ brand->generic entries

API:
  normalize(name: str) -> NormResult
  normalize_list(names: list[str]) -> list[NormResult]
  avg_confidence(results: list[NormResult]) -> float

NormResult dataclass:
  .generic        : INN/generic name
  .brand_matched  : matched brand name (if applicable)
  .confidence     : 0.0-1.0 confidence score
  .match_type     : 'exact'|'alias'|'fuzzy'|'generic'|'not_found'
  .notes          : human-readable explanation

Confidence tiers:
  exact match   = 0.97  (Panadol -> Paracetamol)
  generic name  = 0.95  (input is already a generic INN)
  alias/prefix  = 0.82  (prefix match >= 5 chars)
  fuzzy >= 0.78 = 0.50-0.91 (SequenceMatcher ratio-scaled)
  fuzzy >= 0.62 = 0.35-0.57 (low confidence, warn)
  not_found     = 0.30  (pass-through, name kept as-is)

Catalog categories (200+ entries):
  Antibiotics: Augmentin, Hi-biotic, Ciproxin, Flagyl,
               Zithromax, Klacid, Keflex, Zinnat, etc.
  GI: Colovatil, Antinal, Nexium, Losec, Motilium, etc.
  Analgesics: Panadol, Brufen, Voltaren, Celebrex, Tramal, etc.
  Cardiovascular: Norvasc, Concor, Zestril, Cozaar, Plavix,
                  Warfant, Lipitor, Crestor, Clexane, etc.
  Diabetes: Glucophage, Amaryl, Diamicron, Jardiance, Lantus, etc.
  Thyroid: Euthyrox, Eltroxin, Neomercazole, etc.
  Asthma/COPD: Ventolin, Seretide, Symbicort, Singulair, etc.
  Psychiatric/Neuro: Zoloft, Cipralex, Valium, Risperdal,

SECTION 4  SESSION 3 CHANGES  Drug Lookup Full Rewrite


Feature 1: drug_lookup.py -- Complete Rewrite

Previous state: lookup_drug_info() only queried local SQLite/JSON.
Returned stub "Drug not found" for anything not in local DB.
Never called any external API.

New 3-tier architecture in lookup_drug_info(drug_name):

  Step 0: Normalize brand -> generic via drug_normalizer
  Tier 1: Local drug_db (SQLite + knowledge_base/drugs.json)
          - If structured clinical fields exist, return immediately
          - If local entry found but lacks clinical fields (overview-
            only blob), silently enrich from OpenFDA then return
            with _source="local" (confirmed generic name preserved)
  Tier 2: OpenFDA drug label API
          - _openfda_lookup(name): searches openfda.generic_name
            then openfda.brand_name
          - KEY: urllib.parse.urlencode({"search": f'field:"{name}"'})
            -- do NOT pre-encode quotes (causes double-encoding bug)
          - Validates result: returned label's names must contain
            the searched drug name (prevents wrong matches)
          - INN->USAN retry: if name not found, checks _INN_TO_USAN
            dict and retries with US equivalent (e.g. Paracetamol
            -> Acetaminophen, Salbutamol -> Albuterol)
  Tier 3: RxNorm API
          - CUI lookup -> RxTerms allinfo -> drug class

  _source field: "local" | "openfda" | "rxnorm" | "none"

Feature 2: app.py Source Badge

  Drug profile header now shows source badge:
  - " OpenFDA"  / " Local DB" / " RxNorm"
  Reads info.get("_source","") -- blue pill badge (#E0F4FF/#0277BD)

Feature 3: _parse_openfda_label() -- Rx + OTC Support

  OpenFDA returns two label types. Both now handled:

  Rx labels (branded/prescription):
    mechanism_of_action / description / clinical_pharmacology
    contraindications / adverse_reactions
    dosage_and_administration
    pregnancy / teratogenic_effects / use_in_specific_populations

  OTC labels (Drug Facts format -- e.g. Acetaminophen/Paracetamol):
    purpose         -> mechanism fallback + drug_class fallback
    do_not_use      -> contraindications (merged with Rx field)
    warnings        -> side_effects (merged with adverse_reactions)
    stop_use        -> side_effects (merged)
    dosage_and_administration -> dosage (same field)
    pregnancy_or_breast_feeding -> pregnancy_category fallback

Feature 4: Drug Class Fallback via RxNorm CUI/DAILYMED

  If OpenFDA label has no pharm_class_epc/cs/moa fields,
  _parse_openfda_label() now falls back to RxNorm:
    1. GET rxcui.json?name={generic}&search=2  -> CUI
    2. GET rxclass/class/byRxcui.json?rxcui={cui}&relaSource=DAILYMED
    3. Returns className e.g. "Adrenergic beta-Antagonists"

Feature 5: Section Header Stripping in _clean()

  FDA labels embed section numbers in text fields:
  "12.1 Mechanism of Action Metoprolol is a beta1-selective..."
  "4 CONTRAINDICATIONS Metoprolol tartrate tablets are..."

  _clean() now strips:
  - Numbered Rx section headers: regex for known section names
    (Mechanism of Action, Contraindications, Adverse Reactions,
     Dosage and Administration, Clinical Pharmacology, etc.)
  - OTC Drug Facts headers: Purpose, Uses, Directions, Warnings,
    Do not use, Stop use, Ask a doctor

Feature 6: INN->USAN Mapping (_INN_TO_USAN dict)

  OpenFDA uses US drug names (USAN). International INN names
  must be mapped before querying:
    paracetamol   -> acetaminophen
    salbutamol    -> albuterol
    adrenaline    -> epinephrine
    noradrenaline -> norepinephrine
    suxamethonium -> succinylcholine
    lignocaine    -> lidocaine
    frusemide     -> furosemide
    glibenclamide -> glyburide
    pethidine     -> meperidine
    amfetamine    -> amphetamine

Verified working examples:
  lookup_drug_info('Metoprolol'):
    _source: openfda
    generic_name: Metoprolol Tartrate
    drug_class: Adrenergic beta-Antagonists  (via RxNorm DAILYMED)
    mechanism: "Metoprolol is a beta1-selective (cardioselective)..."
    indications: [real clinical indications from label]
    contraindications: [severe bradycardia, heart block, ...]
    side_effects: [tiredness, dizziness, bradycardia, ...]
    dosage: "Administer once daily with food or after a meal..."

  lookup_drug_info('Panadol'):
    _source: local   (brand normalized: Panadol -> Paracetamol)
    generic_name: Paracetamol
    drug_class: Pain reliever/fever reducer  (from OTC purpose field)
    mechanism: "Pain reliever/fever reducer"
    indications: ["Temporarily relieves minor aches and pains..."]
    contraindications: ["If you are allergic to acetaminophen..."]
    side_effects: ["Liver warning: This product contains..."]
    dosage: "do not take more than directed adults and children..."
    pregnancy_category: "See label"

Module 1: OpenFDA SQLite Cache (Performance)

Target files: database.py, drug_lookup.py

A) database.py -- new openfda_cache table added to _DDL schema:

   CREATE TABLE IF NOT EXISTS openfda_cache (
       generic_name TEXT PRIMARY KEY,
       profile_json TEXT NOT NULL,
       timestamp    DATETIME DEFAULT CURRENT_TIMESTAMP
   );

   - Idempotent: CREATE TABLE IF NOT EXISTS, safe on every startup
   - PRIMARY KEY on generic_name.lower() for O(1) lookup
   - timestamp used to enforce 30-day TTL

B) drug_lookup.py -- two new cache helper functions:

   _cache_get(name: str) -> dict | None
   - Queries openfda_cache WHERE generic_name = ? AND timestamp
     within last 30 days
   - On hit: deserializes JSON, sets _source = "openfda_cached"
   - Returns None on miss or any DB error (fails silently)

   _cache_put(name: str, profile: dict) -> None
   - INSERT OR REPLACE into openfda_cache with JSON-serialized profile
   - Fails silently on any exception

C) drug_lookup.py -- _openfda_lookup() updated:
   - Step 1: _cache_get(name_l) BEFORE any HTTP request
   - Step 2: on successful API parse -> _cache_put(name_l, result)
   - _source values: "openfda" (fresh) | "openfda_cached" (from DB)

   Performance effect: repeat lookups (same drug searched twice)
   return instantly from SQLite instead of calling api.fda.gov.
   Cache valid for 30 days per drug name.


Module 2: Pediatric Dosage Validator (Clinical Safety)

Target files: dosing_validator.py, ocr.py, app.py

A) dosing_validator.py -- new function check_pediatric_dose():

   check_pediatric_dose(drug_generic_name: str,
                        extracted_dosage_str: str,
                        patient_weight_kg: float) -> str | None

   Logic:
   - Normalises drug name via existing _normalise()
   - Extracts mg value: re.search(r"(\d+(?:\.\d+)?)\s*mg", ...)
   - Rule: paracetamol / acetaminophen -> max 15 mg/kg per dose
   - If dose_mg > 15 * weight_kg: returns warning string with
     exact numbers (e.g. "...exceeds 300 mg for 20.0 kg patient")
   - Returns None if dose is safe or mg value not parseable

   Example:
     check_pediatric_dose('paracetamol', '600 mg', 20.0)
     -> "PEDIATRIC OVERDOSE WARNING: paracetamol 600 mg exceeds
        maximum safe pediatric dose (300 mg for 20.0 kg patient;
        limit 15 mg/kg per dose). Reduce dose immediately."

     check_pediatric_dose('paracetamol', '200 mg', 20.0)
     -> None  (safe: 200 < 300)

B) ocr.py -- analyze_prescription_safety() signature extended:

   def analyze_prescription_safety(
       parsed_meds: list,
       patient: str = "",
       prescriber: str = "",
       patient_weight_kg: float = 0.0   # NEW
   ) -> dict:

   After Groq returns and setdefaults run, iterates parsed_meds:
     from dosing_validator import check_pediatric_dose as _cpd
     warn = _cpd(med_name, med_dosage, patient_weight_kg)
     if warn: appends to result["dosing_errors"] with severity="major"
   
   Resulting dosing_error dict:
   {
     "drug": "<generic_name>",
     "prescribed_dose": "<dosage string>",
     "safe_range": "max 300 mg (15 mg/kg for 20.0 kg)",
     "severity": "major",
     "recommendation": "<full warning string>"
   }

C) app.py -- Patient Details section extended:

       "⚖️ Patient Weight (kg)",
       min_value=2.0,  max_value=150.0,
       value=20.0,     step=0.5,

       help="Used for pediatric weight-based dosage safety checks"

   - Session state key: "patient_weight_kg" (float, default 20.0)
   - Value passed as patient_weight_kg= in both analyze calls

Module 3: Strict Administration Rules Engine (Override LLM)

Target file: ocr.py (inside analyze_prescription_safety)

After Groq result and setdefaults, before return:

   _STRICT_RULES = {
       "cholecalciferol": "MUST take with a fatty meal.",
       "vitamin d3":      "MUST take with a fatty meal.",
       "cyproheptadine":  "MUST take 15-30 mins BEFORE meals.",
   }

For each medication in parsed_meds:
  1. Normalise name: (generic_name or name).lower().strip()
  2. Check if any rule key appears IN the normalised name
     (substring match, so "vitamin d3 drops" still matches)
  3. If match found:
     a. Search existing frequency_alerts for that drug
        - If found: OVERWRITE _fa_item["meal_timing"] with rule value
        - If not found: APPEND a new frequency_alert entry
  4. Break after first matching rule (one rule per drug)

New frequency_alert structure when injected by strict rules:
   {
     "drug": "<name>",
     "prescribed_frequency": "<freq from med or N/A>",
     "standard_frequency": "As prescribed",
     "meal_timing": "MUST take with a fatty meal.",
     "severity": "moderate",
     "recommendation": "MUST take with a fatty meal."
   }

Design intent: LLM may return "take with food" for cholecalciferol
but the pharmacologically correct instruction is specifically
"fatty meal" (fat is required for fat-soluble vitamin absorption).
This engine guarantees the correct instruction regardless of what
Groq returns.


Module 4: Low-Confidence Manual Override (Human-in-the-Loop)

Target file: app.py (Prescription Scanner UI)

A) New session state variables (initialized at startup):
   st.session_state.show_low_conf_review = False
   st.session_state.patient_weight_kg    = 20.0

B) "Detect Errors & Interactions" button -- new flow:

   OLD FLOW:
     click -> spinner -> analyze_prescription_safety() -> rerun

   NEW FLOW:
     click -> check name_confidence < 0.75 per medication
       IF any low-confidence drugs found:
         set show_low_conf_review = True -> rerun
         (pipeline PAUSED -- no analysis yet)
       IF all drugs high-confidence:
         set show_low_conf_review = False
         -> analyze_prescription_safety(weight=patient_weight_kg)
         -> safety_result -> rerun

C) Low-confidence review UI (rendered when show_low_conf_review):

   st.warning("⚠️ Low confidence in reading some drug names.
               Please verify before running safety analysis.")

   For each medication where name_confidence < 0.75:
     _cands = med["name_candidates"] or []
     if original name not in _cands: prepend it
     _cands = [c for c in _cands if c] or ["Unknown"]
     confidence_pct = int(round(name_confidence * 100))
     
     st.selectbox(
         f"Medication {i+1} (confidence: {pct}%) — select correct name:",
         options=_cands,
         key=f"low_conf_select_{i}",
     )

   st.button("✅ Confirm & Proceed to Safety Analysis",
             type="primary", key="low_conf_confirm_btn")

D) Confirm button action:
   - Reads all low_conf_select_{i} values from session_state
   - Writes selected name back into raw_json medications[i]["name"]
   - Calls analyze_prescription_safety() with corrected meds list
     and patient_weight_kg
   - Sets show_low_conf_review = False
   - Stores result in safety_result -> rerun

   High-confidence flow is completely unchanged; low-confidence
   flow adds exactly one manual confirmation step.
